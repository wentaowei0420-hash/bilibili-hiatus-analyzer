## 报告安全漏洞

我们非常重视项目的安全性。如果你发现了安全漏洞，请负责任地披露。

### 报告流程

**请勿公开披露安全漏洞！**

如果你发现了安全问题，请通过以下方式私下报告：

1. **GitHub Security Advisory**（推荐）
   - 访问项目的 [Security Advisory](https://github.com/DAILtech/bilibili-hiatus-analyzer/security/advisories/new)
   - 填写漏洞详情
   - 我们会在24-48小时内回复

2. **Email报告**
   - 发送邮件到：security@yourproject.com
   - 标题：[SECURITY] 简短描述
   - 内容包括：
     - 漏洞描述
     - 影响范围
     - 复现步骤
     - 可能的修复方案（如果有）

### 报告内容

一份好的安全报告应包括：

- **漏洞类型**：例如 XSS、CSRF、代码注入等
- **影响范围**：哪些版本受影响
- **漏洞描述**：详细说明问题
- **复现步骤**：如何触发该漏洞
- **影响评估**：可能造成的危害
- **修复建议**：如果你有建议的话

### 响应时间

- **初步响应**：24-48小时内
- **漏洞确认**：3-5工作日
- **修复发布**：根据严重程度，7-30天

### 漏洞等级

我们使用CVSS v3.1评估漏洞严重程度：

- **严重（Critical）**: CVSS 9.0-10.0
  - 立即修复，1-7天内发布补丁
  
- **高危（High）**: CVSS 7.0-8.9
  - 优先修复，7-14天内发布补丁
  
- **中危（Medium）**: CVSS 4.0-6.9
  - 正常修复，14-30天内发布补丁
  
- **低危（Low）**: CVSS 0.1-3.9
  - 常规更新中修复

## 安全最佳实践

### 对于使用者

1. **Cookie安全**
   - ⚠️ 永远不要分享你的Cookie
   - ⚠️ 不要在公共电脑上保存Cookie
   - ⚠️ 定期更换Cookie（重新登录）
   - ⚠️ 不要将Cookie提交到Git仓库

2. **代码安全**
   - ✅ 从官方仓库下载代码
   - ✅ 验证代码完整性
   - ✅ 审查代码再运行（如果你懂技术）
   - ✅ 使用最新版本

3. **运行环境**
   - ✅ 在虚拟环境中运行
   - ✅ 限制脚本权限
   - ✅ 不要以管理员权限运行

4. **数据安全**
   - ⚠️ 输出的CSV可能包含个人信息，妥善保管
   - ⚠️ 不要分享包含UID的数据文件

### 对于开发者

1. **代码审查**
   - 所有PR需要经过审查
   - 关注安全相关的更改
   - 使用静态分析工具

2. **依赖管理**
   - 定期更新依赖
   - 使用已知安全的库
   - 监控依赖的安全漏洞

3. **敏感信息**
   - 永远不要硬编码敏感信息
   - 使用环境变量
   - 不要在日志中输出敏感数据

## 已知安全考虑

### Cookie存储

**问题**：当前版本要求在脚本中硬编码Cookie

**风险**：
- Cookie可能被意外分享
- Cookie可能被提交到版本控制

**缓解措施**：
- `.gitignore`已配置忽略可能包含Cookie的文件
- 文档中强调Cookie安全性
- 未来版本将支持环境变量

**建议**：
```python
# 推荐做法：使用环境变量
import os
COOKIE = os.getenv('BILIBILI_COOKIE', '')
```

```bash
# 运行前设置环境变量
export BILIBILI_COOKIE="your_cookie_here"
python bilibili_hiatus_analyzer.py
```

### 网络请求

**问题**：脚本会向B站服务器发送请求

**风险**：
- 中间人攻击（理论风险）
- Cookie泄露

**缓解措施**：
- 使用HTTPS
- 验证SSL证书
- 不在不安全网络上运行

### 数据输出

**问题**：CSV文件包含个人关注数据

**风险**：
- 暴露个人兴趣
- UP主信息泄露

**缓解措施**：
- `.gitignore`已配置忽略CSV文件
- 文档中提醒数据隐私
- 用户自行保管输出文件
