# 🎯 B站催更分析器

一个自动分析B站关注UP主更新情况的Python脚本，帮你找出最久没更新视频的"鸽王"。

## ✨ 功能特性

- 🔐 使用Cookie模拟登录，自动获取关注列表
- 📊 分析所有关注UP主的最新视频发布时间
- 🏆 生成"鸽王"排行榜，找出最久没更新的UP主
- 💾 自动保存数据为CSV文件，方便查看和分析
- 🛡️ 完善的错误处理机制
- ⏱️ 遵守君子协议，避免请求过快被封禁

## 📋 数据字段

脚本会提取以下信息：

- **UP主姓名**: UP主的昵称
- **UP主UID**: UP主的唯一ID
- **最新视频标题**: 最新发布视频的标题
- **最新视频发布日期**: 视频发布的具体时间
- **未更新天数**: 距离上次发布视频已经过去多少天
- **最新视频播放量**: 该视频的播放次数
- **视频链接**: 视频的B站直达链接

## 🚀 快速开始

### 1. 安装依赖

```bash
pip install -r requirements.txt
```

### 2. 获取B站Cookie

这是最关键的一步！请按照以下步骤获取你的Cookie：

#### 方法一：Chrome浏览器

1. 打开 [B站首页](https://www.bilibili.com) 并登录你的账号
2. 按 `F12` 打开开发者工具
3. 切换到 **Network（网络）** 标签
4. 刷新页面（F5）
5. 在请求列表中点击任意一个请求（建议选择第一个）
6. 在右侧找到 **Request Headers（请求头）**
7. 找到 **Cookie** 字段，点击后面的值，右键选择 **Copy value（复制值）**

#### 方法二：Firefox浏览器

1. 打开 [B站首页](https://www.bilibili.com) 并登录你的账号
2. 按 `F12` 打开开发者工具
3. 切换到 **网络** 标签
4. 刷新页面（F5）
5. 点击任意一个请求
6. 在右侧找到 **请求头**
7. 找到 **Cookie**，复制完整的值

#### Cookie示例（仅供参考格式）
```
SESSDATA=xxxxx; bili_jct=xxxxx; DedeUserID=xxxxx; ...
```

⚠️ **重要提示**：
- Cookie包含你的登录凭证，**请勿泄露给他人**
- Cookie会过期，如果脚本提示登录失败，请重新获取
- 复制完整的Cookie字符串，不要遗漏任何部分

### 3. 配置脚本

打开 `bilibili_hiatus_analyzer.py`，找到配置区域：

```python
# ===========================
# 配置区域 - 请在这里填入你的信息
# ===========================

# Cookie配置
COOKIE = "在这里粘贴你的Cookie"
```

将刚才复制的Cookie粘贴到引号中：

```python
COOKIE = "SESSDATA=xxxxx; bili_jct=xxxxx; DedeUserID=xxxxx; ..."
```

### 4. 运行脚本

```bash
python bilibili_hiatus_analyzer.py
```

### 5. 查看结果

脚本运行完成后会：
1. 在控制台显示Top 10"鸽王"排行榜
2. 生成 `bilibili_hiatus_ranking.csv` 文件，包含所有UP主的详细数据

用Excel或其他表格软件打开CSV文件，即可查看完整的排行榜。

## ⚙️ 高级配置

你可以在脚本中修改以下参数：

```python
# 输出文件名
OUTPUT_CSV = "bilibili_hiatus_ranking.csv"

# 请求延时（秒）- 遵守君子协议
REQUEST_DELAY = 1
```

- `OUTPUT_CSV`: 修改输出的CSV文件名
- `REQUEST_DELAY`: 调整每次请求之间的延时（建议至少1秒）

## ❗ 常见问题

### Q1: 提示"Cookie可能已过期"怎么办？

**A**: Cookie有时效性，通常几天到几周会过期。请重新按照步骤获取新的Cookie。

### Q2: 提示"网络请求失败"怎么办？

**A**: 可能是网络问题或B站服务器暂时无响应。可以：
- 检查网络连接
- 稍后再试
- 增加 `REQUEST_DELAY` 的值

### Q3: 某些UP主显示"暂无视频"？

**A**: 可能的原因：
- UP主确实没有发布过视频（只发动态）
- UP主删除了所有视频
- UP主的视频被设为私密或隐藏

### Q4: 运行速度很慢？

**A**: 这是正常现象。为了避免被B站封禁，脚本会在每次请求之间延时1秒。如果你关注了200位UP主，大约需要3-4分钟才能完成。

### Q5: CSV文件中文乱码？

**A**: 脚本使用 `utf-8-sig` 编码保存，Excel应该能正确显示。如果仍有问题：
- Windows Excel: 右键文件 → 打开方式 → 记事本 → 另存为 → 编码选择"ANSI"
- 或使用WPS Office、Google Sheets等软件打开
